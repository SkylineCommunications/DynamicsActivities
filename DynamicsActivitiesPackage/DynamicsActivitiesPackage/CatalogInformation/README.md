# DynamicsActivitiesPackage

![WIP](./Images/wip.png)

## Overview

DynamicsActivitiesPackage deploys the **DynamicsActivities** React web application to a DataMiner Agent as part of a `.dmapp` package.

On installation, the web app is placed at:

`C:\Skyline DataMiner\Webpages\public\DynamicsActivities`

This allows hosting the app through DataMiner webpages (for example `/public/DynamicsActivities/`).

## What is included

The package includes the static web build generated by Vite from `dist-dataminer`, including:

- `index.html`
- `web.config`
- `assets/*` (hashed JavaScript/CSS bundle files)

These files are added as DataMiner companion files under:

`PackageContent/CompanionFiles/Skyline DataMiner/Webpages/public/DynamicsActivities`

## Build and packaging

1. Build the web app:
   - `npm install`
   - `npm run build`
2. Build the `DynamicsActivitiesPackage` project.

If `dist-dataminer` is not present when building the package project, the project will automatically run npm install and build to generate it.

## Output artifact

The generated DataMiner package is available under:

`bin/<Configuration>/net48/DataMinerBuild/DynamicsActivitiesPackage.<version>.dmapp`

Example:

`bin/Debug/net48/DataMinerBuild/DynamicsActivitiesPackage.1.0.0.dmapp`

## Prerequisites

- DataMiner Agent version compatible with package requirement (`10.3.0.0 - 12752` or higher).
- Node.js/npm available on the build machine when the web output must be generated.
