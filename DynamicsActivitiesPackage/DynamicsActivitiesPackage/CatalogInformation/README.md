# DynamicsActivitiesPackage

## About

This package deploys the **DynamicsActivities** React web application to a DataMiner Agent as part of a `.dmapp` package.

On installation, the web app is placed in the `C:\Skyline DataMiner\Webpages\public\DynamicsActivities` folder. This allows hosting the app through DataMiner webpages (for example `/public/DynamicsActivities/`).

## Package components

The package includes the static web build generated by Vite from `dist-dataminer`, including:

- `index.html`
- `web.config`
- `assets/*` (hashed JavaScript/CSS bundle files)

These files are added as DataMiner companion files in the `PackageContent/CompanionFiles/Skyline DataMiner/Webpages/public/DynamicsActivities` folder.

## Building and packaging

1. Build the web app:

   - `npm install`
   - `npm run build`

1. Build the `DynamicsActivitiesPackage` project.

   If `dist-dataminer` is not present when building the package project, the project will automatically run `npm install` and `npm run build` to generate it.

## Output artifact

The generated DataMiner package will be available in the following folder: `bin/<Configuration>/net48/DataMinerBuild/DynamicsActivitiesPackage.<version>.dmapp`

Example: `bin/Debug/net48/DataMinerBuild/DynamicsActivitiesPackage.1.0.0.dmapp`

## Prerequisites

- DataMiner Agent with version `10.3.0.0 - 12752` or higher
- Node.js/npm available on the build machine when the web output must be generated.
